# AI Image Analysis Platform (SIS Big Data)

Данный проект представляет собой платформу для интеллектуального анализа изображений с использованием искусственного интеллекта. Пользователи могут загружать изображения, которые затем автоматически обрабатываются нейросетью (Google Gemini), а результаты анализа сохраняются в базе данных и доступны для просмотра.

## 🚀 Системный дизайн и логика проекта

Проект построен на базе бессерверной архитектуры (Serverless) с использованием сервисов Firebase. Это обеспечивает высокую масштабируемость и надежность.

### Логика работы (User Flow):
1. **Загрузка:** Пользователь авторизуется в системе и загружает изображение (drag-and-drop или через выбор файла) через React-интерфейс.
2. **Хранение:** Файл напрямую отправляется в **Firebase Storage** в изолированную папку пользователя (`uploads/{userId}/{fileName}`).
3. **Триггер (Событие):** Загрузка файла в Storage автоматически инициирует событие `onObjectFinalized`, которое вызывает облачную функцию (Firebase Function).
4. **Обработка (Backend):** 
   - Облачная функция создает запись в **Firestore** со статусом `processing`.
   - Скачивает изображение во временную память.
   - Отправляет изображение на анализ в **Google Gemini AI** через `@google/genai`.
5. **Сохранение результатов:** После получения ответа от ИИ, функция обновляет документ в **Firestore**, меняя статус на `completed` и записывая результаты анализа.
6. **Отображение:** Клиентское приложение (Frontend) может в реальном времени получать обновления из Firestore и показывать результат пользователю.

## 🛠 Технологический стек

### Frontend (Клиентская часть)
- **Фреймворк:** React 19
- **Сборщик:** Vite
- **Язык:** TypeScript
- **Стилизация:** CSS / Glassmorphism UI
- **Иконки:** Lucide React
- **Интеграция:** Firebase Client SDK (Auth, Storage, Firestore)

### Backend (Серверная часть)
- **Среда выполнения:** django
- **Фреймворк:** Firebase Cloud Functions (v2)
- **Язык:** TypeScript
- **База данных:** Cloud Firestore (NoSQL)
- **Хранилище:** Firebase Storage
- **ИИ Интеграция:** Google GenAI SDK (`@google/genai`)

## 🔒 Безопасность
- **Изоляция данных:** Изображения каждого пользователя сохраняются в индивидуальные папки (`uploads/{userId}/`).
- **Storage Rules:** Настроены строгие правила безопасности Firebase Storage, разрешающие пользователям доступ только к их собственным файлам.
- **Firestore Rules:** Разделение доступа к документам аналитики (каждый видит только свои запросы).
